# Widgets Examples (Semaine 1)

Bienvenue ! Ce projet est un **bac à sable de widgets** pour la semaine 1.  
L'objectif est simple : **comprendre la structure d'une app Flutter** et **jouer avec les widgets** sans se perdre dans le reste.

## À quoi vous devez faire attention

Pour le moment, **concentrez-vous uniquement sur `lib/widgets/`** :  
c’est là que se trouvent les exemples de widgets (Row, Column, Card, Stack, etc.).

## Ce que vous pouvez ignorer pour l’instant (mais à quoi ça sert)

Vous verrez d'autres dossiers (android, ios, web, test, etc.).  
**Ne vous inquiétez pas : on n’y touche pas encore.**  
Ils sont là pour **mettre en place la mini‑app** et permettre de **l’exécuter sur émulateur ou téléphone physique**, mais **ce n’est pas le sujet de la semaine 1**.

Petit repère rapide :
- `lib/main.dart` : point d’entrée de l’app
- `lib/screens/` : écrans qui regroupent plusieurs widgets
- `lib/previews/` : prévisualisation rapide des widgets
- `test/`, `android/`, `ios/`, `web/` : parties techniques pour compiler et tester

## Comment naviguer rapidement

Si vous êtes un peu perdu, voici un chemin simple :
1) Ouvrez `lib/widgets/` pour apprendre chaque widget séparément  
2) (Optionnel) Regardez `lib/screens/` pour voir des exemples complets  
3) (Optionnel) Ouvrez `lib/main.dart` pour comprendre le point de départ  

L’idée est d’apprendre **par petits morceaux**, pas tout d’un coup.

## Prévisualiser dans VS Code

Vous pouvez **prévisualiser tous ces widgets directement dans VS Code** grâce à l’outil de preview.  
Ouvrez `lib/previews/widget_previews.dart`, puis lancez la prévisualisation pour voir l’ensemble des widgets sans exécuter toute l’app.

Bon apprentissage !
